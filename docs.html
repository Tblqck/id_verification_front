Docs

One endpoint. Three verification modes.

The core of the API is a single multipart POST /api/v1/verify call. Send a document image plus a selfie, a holding photo, or liveness frames — get back a structured verdict.

Authentication

API key header

Every request to /api/v1/verify, /api/v1/save and /api/v1/mrz requires an X-Api-Key header. Keys are issued once your account is approved — request access to get one.

X-Api-Key: <your-api-key>
POST /api/v1/verify

Run a verification.

Multipart form fields. mode selects which second input you provide alongside the ID document.

Required fields

country      ISO alpha-2 code, e.g. "GB"
doc_type     "passport" | "national_id" | "drivers_license" | "residence_permit"
id_image     the captured ID document image
mode         1 = ID + selfie
             2 = ID + photo holding the ID
             3 = ID + up to 5 liveness frames

Example — mode 1 (ID + selfie)

curl -X POST https://api.idntory.com/api/v1/verify \
  -H "X-Api-Key: $IDNTORY_API_KEY" \
  -F "country=NG" \
  -F "doc_type=passport" \
  -F "mode=1" \
  -F "id_image=@id_front.jpg" \
  -F "selfie=@selfie.jpg"

Response

A JSON report with a per-module breakdown and an overall verdict.

{
  "overall_verdict": "pass",
  "overall_score": 0.91,
  "document": { "verdict": "pass", "score": 0.94, "refs_checked": 3 },
  "face":     { "verdict": "match", "score": 0.88 },
  "liveness": { "is_live": true, "score": 0.93 },
  "mrz":      { "fields": { "doc_number": "...", "birth_date": "..." } }
}
